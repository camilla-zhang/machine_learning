# Helper function for plotting top positive and negative coefficients

def visualize_coefficients(coefficients, feature_names, n_top_features=25):
    """Visualize coefficients of a linear model.
    Parameters
    ----------
    coefficients : nd-array, shape (n_features,)
        Model coefficients.
    feature_names : list or nd-array of strings, shape (n_features,)
        Feature names for labeling the coefficients.
    n_top_features : int, default=25
        How many features to show. The function will show the largest (most
        positive) and smallest (most negative)  n_top_features coefficients,
        for a total of 2 * n_top_features coefficients.
    """
    coefficients = coefficients.squeeze()
    if coefficients.ndim > 1:
        raise ValueError("coeffients must be 1d array or column vector, got"
                         " shape {}".format(coefficients.shape))
    coefficients = coefficients.ravel()

    if len(coefficients) != len(feature_names):
        raise ValueError("Number of coefficients {} doesn't match number of"
                         "feature names {}.".format(len(coefficients),
                                                    len(feature_names)))
    # get coefficients with large absolute values
    coef = coefficients.ravel()
    positive_coefficients = np.argsort(coef)[-n_top_features:]
    negative_coefficients = np.argsort(coef)[:n_top_features]
    interesting_coefficients = np.hstack([negative_coefficients,
                                          positive_coefficients])
    # plot coefficients
    plt.figure(figsize=(15, 5))
    colors = ['#ff2020' if c < 0 else '#0000aa'
              for c in coef[interesting_coefficients]]
    plt.bar(np.arange(2 * n_top_features), coef[interesting_coefficients],
            color=colors)
    feature_names = np.array(feature_names)
    plt.subplots_adjust(bottom=0.3)
    plt.xticks(np.arange(1, 1 + 2 * n_top_features),
               feature_names[interesting_coefficients], rotation=60,
               ha="right")
    plt.ylabel("Coefficient magnitude")
    plt.xlabel("Feature")

#i. Set up custom tokenizer class. Using LemmaTokenizer but can also use Stemming
class LemmaTokenizer(object):
    def __init__(self):
        self.wnl = WordNetLemmatizer()
    def __call__(self, doc):
        return [self.wnl.lemmatize(t) for t in word_tokenize(doc)]

#iii. Define the count vectorizer w/the custom tokenizer, then fit_transform it

# Function for creating model with 2 hidden layers
def create_model(hiddenunits):
  model = Sequential() #Lay out architecture of model with Sequential
  model.add(Dense(hiddenunits, input_dim = 4, activation='relu')) #Hidden layer 1: set input_dim to # of features (4 features)
  model.add(Dense(hiddenunits, activation='relu')) #Hidden layer 2
  model.add(Dense(3, activation='softmax')) #Output layer: 3 DV classes, change activation based on # of DV classes. OMIT IF YOU ARE PERFORMING REGRESSION OVER CLASSIFIER
# Compile model
  model.compile(loss='categorical_crossentropy', optimizer='sgd', metrics=['accuracy'])
  return model

model = KerasClassifier(build_fn=create_model, epochs=20, verbose=0, batch_size=32)

*Based on grid search CV, the best model was the neural network with an optimal 69 hidden units. This model had a mean CV score of 0.997 and a predictive test score of 0.987, which is a strong accuracy score.

I then compared this model with two others using predictive accuracy scores - one with 10 hidden units and another with 60 hidden units. Both models had a lower prediction score score, with the 60 hidden units having a higher score at 0.60, and the 10 units having a slightly lower score at 0.58. Since the model with 38 hidden units produced the highest scores of the three models, I chose that to be my final model.

The choice of how many hidden nodes is appropriate (outside of gridsearch CV) lies in the trade off between training error and test error (over-fitting vs underfitting). When you have more hidden neurons, you are capturing more granular relationships within the data and have more model complexity. At some point, however, as you increase the number of hidden neurons, the prediction (testing) error increases due to the complexity of the model.Once we go into that territory, we end up overfitting on the model and providing a worse application to other data sets outside the training set.*
